# Security Policy

## Official Sources

- Official GitHub organization: [github.com/Kishu-Community](https://github.com/Kishu-Community)
- Official homepage: [kishu.finance](https://kishu.finance)
- Official KISHU token tracker: [Etherscan](https://etherscan.io/token/0xa2b4c0af19cc16a6cfacce81f192b024d625817d)
- Official KISHU token contract: `0xa2b4c0af19cc16a6cfacce81f192b024d625817d`

Public GitHub repositories can be forked, cloned, copied, and rebranded by anyone. Treat forks, lookalike repositories, copied websites, fake support accounts, issue comments, pull requests, and direct messages as untrusted unless they are linked from the official organization or the official homepage.

## Scam And Phishing Warnings

Kishu Community maintainers will never ask for:

- Seed phrases or recovery phrases
- Private keys
- Wallet files
- Remote desktop access
- Up-front payments to recover tokens
- Wallet connection to "verify" holdings
- Contract approvals outside a clearly documented official app flow

If a site or account asks for any of the above, assume it is malicious.

## Reporting

Report suspected phishing, scam copies, malicious forks, fake support accounts, or leaked credentials through a GitHub issue in the relevant official repository when safe to do so. If the report includes sensitive information, do not post secrets publicly. Contact a repository owner directly through GitHub instead.

## Repository Handling

Recovered historical repositories may contain old links, archived code, or abandoned project claims. Read each repository's README and archive notes before treating it as active infrastructure.

Security-sensitive changes should be made through visible commits in the official organization and should reference the reason for the change.
